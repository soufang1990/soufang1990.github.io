我的第一篇 Hexo 博客 | 从搭建到发布，新手也能轻松上手

2026/4/28 博客搭建 Hexo 博客 前端 新手教程

大家好～ 这是我的第一篇 Hexo 博客，也是我第一次用静态博客框架记录自己的学习和开发心得。
作为一名常年和 PHP、JS、MySQL 打交道的开发者，之前一直想拥有一个属于自己的个人博客，用来记录项目踩坑、技术总结，也想沉淀一些自己的思考。对比了 Wordpress、Typecho 等博客系统后，最终选择了 Hexo —— 轻量、高效，纯静态部署，不用操心服务器和数据库，对开发者来说简直太友好了。

为什么选择 Hexo？

其实作为有编程基础的人，搭建任何博客系统都不算难，但 Hexo 打动我的几个点，分享给同样在纠结的朋友：

  • ✅ 轻量高效：基于 Node.js 开发，生成静态页面速度极快，本地预览秒加载，部署后访问也很流畅；
  • ✅ 开发友好：核心是 JS 编写，作为熟悉 JS 的开发者，后续可以轻松修改源码、自定义功能，自由度拉满；
  • ✅ 部署简单：无需数据库，直接将生成的静态文件部署到 GitHub Pages、Gitee 等平台，免费又稳定；
  • ✅ 生态完善：主题、插件丰富，无论是想做技术博客、生活随笔，还是文档站点，都能找到合适的配置。

我的 Hexo 搭建小记

虽然有编程基础，但第一次接触 Hexo 还是走了一点点小弯路，这里简单记录下核心步骤，给新手避坑：

  1. 安装环境：先安装 Node.js（Hexo 运行依赖），再通过 npm 全局安装 Hexo，一行命令就能搞定；
  2. 初始化博客：执行 hexo init 博客名称，自动生成完整的博客目录结构，不用手动配置；
  3. 配置基础信息：修改根目录的 _config.yml，设置博客标题、作者、描述、主题等，简单易懂；
  4. 新建文章：就是现在这一步啦，执行 hexo new “文章标题”，会自动在 source/_posts 目录生成 Markdown 文件；
  5. 本地预览：执行 hexo g 生成静态页面，再执行 hexo s，访问 http://localhost:4000 就能看到效果；
  6. 部署上线：后续配置好 GitHub Pages，执行 hexo d 就能一键部署，让全世界看到自己的博客。
    其实整个过程比我想象中更简单，尤其是有 JS 基础，看 Hexo 的配置和源码都很轻松，哪怕是新手，跟着教程走，半小时也能完成搭建。

博客未来规划

开这个博客，主要是想做两件事：

  1. 技术沉淀：记录 PHP、JS、MySQL 相关的开发技巧、项目踩坑、学习笔记，把零散的知识系统化，也希望能帮到有需要的朋友；
  2. 个人成长：偶尔分享一些学习心得、工具推荐，也记录自己在技术路上的成长，避免半途而废。
    这是第一篇，也是一个开始。后续会慢慢优化博客的样式、添加评论、访问统计等功能，也会持续更新技术相关的内容。
    如果屏幕前的你也在搭建 Hexo 博客，或者有相关的问题，欢迎一起交流探讨～ 毕竟，技术之路，独行快，众行远。
    最后，愿我们都能在自己喜欢的领域，慢慢沉淀，慢慢发光 ✨
